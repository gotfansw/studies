# Pull Request: Создание схемы БД — students / subjects / marks

## Что сделано

Создана база данных из трёх связанных таблиц по схеме с лекции.  
В каждой таблице по **10 записей**. Таблица `marks` связана внешними ключами с обеими таблицами.

---

## Структура таблиц

### `students` — студенты

| Поле       | Тип              | Описание                    |
|------------|------------------|-----------------------------|
| `id`       | INT IDENTITY PK  | Первичный ключ, автоинкремент |
| `name`     | VARCHAR(30) NOT NULL | Имя                     |
| `lastname` | VARCHAR(30) NOT NULL | Фамилия                 |
| `birthday` | DATETIME NULL    | Дата рождения               |

---

### `subjects` — предметы

| Поле    | Тип              | Описание                    |
|---------|------------------|-----------------------------|
| `id`    | INT IDENTITY PK  | Первичный ключ, автоинкремент |
| `name`  | VARCHAR(30) NOT NULL | Название предмета       |
| `hours` | SMALLINT NULL    | Количество часов            |

---

### `marks` — оценки

| Поле      | Тип                        | Описание                              |
|-----------|----------------------------|---------------------------------------|
| `stud_id` | INT, FK → students(id)     | Ссылка на студента                    |
| `subj_id` | INT, FK → **subjects(id)** | **Новое поле** — ссылка на предмет    |
| `ddate`   | DATETIME DEFAULT GETDATE() | Дата выставления оценки               |
| `mark`    | TINYINT CHECK (>1 AND <=5) | Оценка от 2 до 5                      |

> `subj_id` — добавленное поле с внешним ключом к таблице `subjects`.  
> Это связывает каждую оценку с конкретным предметом.

---

## Связи между таблицами

```
students ──< marks >── subjects
   (id)    stud_id    subj_id   (id)
```

- `marks.stud_id` → `students.id` — один студент может иметь много оценок  
- `marks.subj_id` → `subjects.id` — одному предмету может соответствовать много оценок
